# memory.md テンプレート（運用メモ・手動整理用）

> 重要な注意
> Claude Codeには、Claudeが自動で学習内容を書き込む「Auto memory」機能があります（詳細は公式情報を確認してください）。
> この `memory.md` は、Auto memoryとは別に、**人間が手動で整理する運用メモ**として使うテンプレートです。
> 両者を混同しないように運用してください。Auto memoryの具体的な格納場所やファイル名は環境により異なる可能性があるため、公式情報での確認が必要です。

---

## このファイルの使い方

<!--
たとえ話：
- CLAUDE.md = 取扱説明書（恒久的なルール）
- memory.md = 作業中の注意点メモ（今回の仕事で気をつけること）
- skills = 専用レシピ（繰り返し使う手順書）

いつ書くか：
- 作業中に「次回も気をつけたい」と思ったその瞬間
- 失敗した直後（忘れないうちに原因と対策をメモ）
- 上司や取引先から指摘を受けた直後
- 週次・月次の振り返り時

いつ消すか（棚卸し）：
- 書いてから3か月以上使わなかった項目
- CLAUDE.md や skill に昇格させた項目（重複を避けるため）
- 状況が変わって、もう当てはまらなくなった項目
-->

- 記入タイミング：作業中に気づいた瞬間、または週次の振り返り時
- 見直しタイミング：月1回、全項目を棚卸し（「まだ有効か」を確認）
- 削除の目安：3か月以上参照しなかった項目／CLAUDE.md・skillに昇格した項目


## 2. 過去に起きた失敗とその対策

<!--
記入のヒント：
- 「何が起きたか」「なぜ起きたか」「次はどうするか」を3点セットで
- 責任追及ではなく、再発防止のためのログとして書く
- 日付を入れておくと、あとで棚卸ししやすい
-->

記入例：
```
【2026-03-15】元CSVを上書き保存してしまい、復旧に半日
- 原因：作業中に Ctrl+S を押す癖があり、コピーせず編集していた
- 対策：作業開始時に必ず copy コマンドで _work ファイルを作る
- 対策：CLAUDE.md の「絶対に変更してはいけないファイル」に明記済み

【2026-04-02】前月比の計算でゼロ除算エラー
- 原因：前月が0だった新規商材を計算対象に含めていた
- 対策：前月=0 の行はスキップし、「新規」フラグを立てる
```

（ここに記入）

## 3. 好み・文体の確定事項

<!--
記入のヒント：
- 自分の好み、上司の好み、取引先の好みを明文化
- 「こう書いてほしい」「こう書かないでほしい」の両方
- 抽象的な好みではなく、具体例を添える
-->

記入例：
```
- 見出しは全角文字で統一（英数字は使うが、記号装飾は使わない）
- 箇条書きの先頭は「・」ではなく「- 」を使う（markdown準拠）
- 上司の鈴木室長は結論ファースト。冒頭3行でサマリーを書く
- 社外向け文書は句点の後に半角スペースを入れない
```

（ここに記入）

## 4. 使ってはいけない表現・用語

<!--
記入のヒント：
- 過去に指摘を受けた表現
- 社内ルールで禁止されている表現
- 取引先に誤解を与える用語
- なぜダメかも簡潔に添える
-->

記入例：
```
- 「なるはや」→ 納期が曖昧なのでNG。必ず日時で指定する
- 「とりあえず」→ ビジネス文書では使わない
- 「確認お願いします」（単独）→ 何を確認すべきか明記する
- 「当社」vs「弊社」→ 社外向けは「弊社」で統一
- 絵文字・顔文字全般（社外文書）
```

（ここに記入）

## 5. 成功したパターン

<!--
記入のヒント：
- 「これはうまくいった」というやり方を記録
- 再現できるように、状況・手順・結果をセットで
- 次に似た状況が来た時、そのまま真似できる粒度で書く
-->

記入例：
```
【レポート冒頭サマリー】
状況：経営会議で「結論が分かりにくい」と指摘された
やったこと：1ページ目を「今月のポイント3つ」だけにした
結果：その後「読みやすくなった」と好評。継続採用

【Claude Code への依頼】
状況：いきなり作業させると意図とずれがちだった
やったこと：依頼文末尾に「作業前に計画だけ提示してください」を固定で入れた
結果：手戻りが激減。毎回このフレーズを入れることにした
```

（ここに記入）

## 7. 古くなったので削除予定（棚卸し欄）

<!--
記入のヒント：
- 「もう使っていない」「状況が変わった」項目をここに一度移す
- 即消さず、1か月保留してから削除するのが安全
- 削除理由を添える（後で「なぜ消した？」と思わないように）
-->

記入例：
```
- 【削除予定 2026-04-01 棚卸し】
  旧営業部の田中さん宛ての依頼ルール
  → 田中さんが異動したため、もう該当しない。5月末に削除

- 【削除予定 2026-04-10 棚卸し】
  CSV 手動インポートの手順メモ
  → 自動連携に移行したため不要。5月末に削除

- 【削除予定 2026-04-24 棚卸し】
  「メール本文は3段落で」ルール
  → CLAUDE.md に昇格させたため、こちらからは削除。5月末に消す
```

（ここに記入）

- [ ] 各項目に日付が入っているか（記入日・見直し期限）
- [ ] 3か月以上参照していない項目はないか
- [ ] CLAUDE.md や skill に昇格させた項目が、ここに残っていないか
- [ ] 「削除予定」欄の項目は、保留期間を過ぎたら本当に削除したか
- [ ] 新しく追加した項目は、どのセクションに入れるか適切か
