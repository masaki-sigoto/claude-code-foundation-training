# Claude Code 基礎勉強会 講師台本（45枚版／90分対応）

> 対象：非エンジニアの業務スタッフ（初心者）
> スライド構成：`docs/02_slides/slide_structure.md`（全45枚版・スライド1〜57の章立てに対応）
> 文体：口語・実際に声に出して読める文体
> 注記：Claude Code の仕様は更新が早い領域です。本台本では公式仕様を断定せず「公式情報で要確認」のスタンスを徹底してください。
> 統一比喩：Claude Code＝机に来てくれるアシスタント／CLAUDE.md＝家事代行さんに渡す家のルール／memory＝何度か来るうちに覚えた相手の好み／skills＝料理のレシピ／MCP＝外の倉庫・別の部屋への鍵／hooks＝玄関を出る前のチェック／subagents＝引っ越しの荷造り担当
> 共通フレームワーク：6ステップ説明型（一言／日常たとえ／仕事たとえ／Claude Codeでは／勘違い／最初の使い方）と「指示文クッション方式」（雑要望→チャットAIで整形→Claude Code、分岐は途中報告→チャットAIで壁打ち→次の指示）

## 冒頭：講師イントロダクション（スライド1〜4／所要6分）

### 章の冒頭の語り（30秒）

> 「みなさん、こんにちは。本日『Claude Code 基礎勉強会』の講師を担当します、○○です。最初の数分は今日のゴールと、扱わない範囲をはっきり共有させてください。一言でいうと、今日は『Claude Code に作業を頼めるようになる』ための基礎の腹落ち会です。」

### スライド1：タイトル

> 「タイトルスライドなのでサッと進めます。今日のテーマは Claude Code、副題は『業務スタッフのための入門編』です。プログラミングが分からない前提でお話しします。私自身もエンジニアではありません。安心して聞いてください。」

【問いかけ】「Claude Code、名前だけは聞いたことがあるという方、軽く手を挙げてもらえますか？」
【日常のたとえ】「初めて行く料理教室の自己紹介と同じで、ここはまだ準備運動です。」
【仕事でのたとえ】「キックオフミーティングの『議題確認』のスライドだと思ってください。」
【ここで混乱しやすい】タイトルスライドで難しい話をしないこと。受講者を安心させるトーンで。
【つなぎ】「次のスライドで今日のゴールを揃えます。」

### スライド2：今日のゴール

> 「今日のゴールは5つです。Claude Code の正体を自分の言葉で説明できる、普通のチャットAIとの違いを言える、CLAUDE.md／memory／skills の違いがざっくり区別できる、雑な要望を整った『指示文』に直す方法がわかる、そして最後に『自分の業務に1つ持ち帰る』。この5つです。全部完璧に覚える必要はありません。聞いたことある、という状態を目指しましょう。」

【問いかけ】「この5つの中で『これが一番気になる』というものはどれですか？ 心の中で1つ決めておいてください。」
【日常のたとえ】「旅行のしおりを最初に開く瞬間に近いです。今日どこに行くかをまず合意します。」
【仕事でのたとえ】「会議の冒頭で『今日のアウトプット』を確認するのと同じです。」
【ここで混乱しやすい】「全部完璧に理解しないと使えない」と思わせないこと。「持ち帰る1個」を強調する。
【つなぎ】「次のスライドで、逆に『今日扱わないこと』も先に共有します。」

### スライド3：今日扱わないこと

> 「今日扱わないことを先にお伝えします。プログラミングの教育はしません。MCP の深掘り設定もしません。個別ツールの細かい設定値も扱いません。公式仕様の暗記もしません。代わりに『概念を腹落ちさせること』に集中します。仕様は更新が早いので、最新は必ず公式情報で確認してください。」

【問いかけ】「『深いところまで知りたかったのに』とがっかりされた方、いますか？ 安心してください、概念さえ掴めれば公式は読めます。」
【日常のたとえ】「車の教習所の最初の日に、エンジン分解の話はしないですよね。今日はその位置付けです。」
【仕事でのたとえ】「新人研修の初日に専門用語の暗記テストはしないのと同じです。」
【ここで混乱しやすい】「結局なにを覚えればいいの？」となりやすいので、「腹落ち」というキーワードを置く。
【つなぎ】「では現場でよく起きる『うまくいかないあるある』から入ります。」

### スライド4：Claude Code を導入したが起きがちなこと

> 「Claude Code を入れた現場でよく起きることを並べました。指示と違うものが出てくる、前回教えたことをまた聞かれる、どこを触ったか分からなくなる、確認なしに作業が進む、結局自分でやった方が早い、と諦める。心当たりある方も多いと思います。今日は『なぜこれが起きるのか』を理解して、原因に手を打てるようになるのがゴールです。」

【問いかけ】「この5つの中で、自分が一番ヒヤッとした、もしくは心当たりがあるのはどれですか？ 隣の方と30秒だけ話してみてください。」
【日常のたとえ】「家事代行を頼んだら、思ってもいない物まで捨てられていた、というあの感じです。」
【仕事でのたとえ】「新人さんに『いい感じにやっといて』と頼んで、出てきた資料が想定と違うあの瞬間です。」
【ここで混乱しやすい】「Claude Code が悪い」という結論にしない。原因は依頼の仕方にあるという伏線を張る。
【つなぎ】「では原因を理解するために、まず『Claude Code とは何者か』から始めます。」

### 章のまとめの語り（20秒）

> 「ここだけ覚えて帰ってください。今日は『腹落ち』が目的、完璧暗記ではない。これだけです。」

## 第1章：Claude Code って何？（スライド5〜10／所要10分）

### 章の冒頭の語り（30秒）

> 「第1章、一言でいうと『Claude Code は机に来て作業してくれるアシスタントです』。普通のチャットAIが『相談相手』なら、Claude Code は『手を動かす作業担当』です。役割が違う、別物だと最初に押さえてください。」

### スライド5：第1章「一言でいうと」

> 「Claude Code を一言で言うと、机に来て作業してくれる AI エージェントです。普通のチャットAIは『相談相手』、Claude Code は『作業してくれる人』。役割が違うので、片方だけ使うのではなく、両方を組み合わせるのが今日の中心メッセージです。」

【問いかけ】「『相談』と『作業』、皆さんの今日のタスクの中ではどちらが多いですか？」
【日常のたとえ】「友達に電話で相談するのが普通のチャットAI、家に来てくれて掃除してくれる人が Claude Code です。」
【仕事でのたとえ】「メール文面の相談相手がチャットAI、その文面で実際にメールを下書き保存してくれるのが Claude Code です。」
【ここで混乱しやすい】「Claude Code は ChatGPT の上位版」と思ってしまう人が多いです。違います、用途が別です。
【つなぎ】「では各社の AI を比較して、Claude Code の位置付けを地図にしましょう。」

### スライド6：ChatGPT/Gemini/Claude の3社比較

> 「3社それぞれに『チャット相談用』と『作業エージェント用』の2系統があります。OpenAI は ChatGPT と Codex、Google は Gemini と Gemini CLI、Anthropic は claude.ai と Claude Code、というように。Claude Code は Anthropic が出している『作業エージェント側』です。サービス名と仕様は更新があるので、最新は各社公式で確認してください。」

【問いかけ】「皆さんの会社で『相談用』として一番使われている AI はどれですか？」
【日常のたとえ】「同じ会社の中に『総合受付』と『現場作業員』がいるイメージです。担当が違います。」
【仕事でのたとえ】「コンサル会社が『提案チーム』と『実装チーム』を持っているのと同じ構造です。」
【ここで混乱しやすい】「Claude Code じゃないとダメ？」ではありません。3社それぞれ作業エージェントを出しています。今日はたまたま Claude Code を扱うだけ、と説明する。
【つなぎ】「では Claude Code を日常のたとえで掴みましょう。」

### スライド7：Claude Code を日常でたとえると

> 「日常でたとえます。チャットAI は電話相談、声だけで家には来ません。Claude Code は家に来てくれるアシスタント。実際に部屋を片付けたり、書類を整理したりします。だからこそ『何をどう触ってよいか』を事前に伝える必要がありますし、終わったら『何をやったか』を必ず確認する。これが基本姿勢です。」

【問いかけ】「家に他人が来て作業する、と聞いて『ちょっと怖い』と感じませんでしたか？ その感覚は正しいです。」
【日常のたとえ】「家事代行さんが『触らない部屋』を知らないまま入ってきたら、ヒヤッとしますよね。」
【仕事でのたとえ】「派遣の方に初日にいきなり全フォルダの権限を渡したら、事故が起きます。」
【ここで混乱しやすい】「便利そう」だけで終わらせない。「だから事前準備が要る」までセットで覚える。
【つなぎ】「次は仕事のたとえに置き換えます。」

### スライド8：Claude Code を仕事でたとえると

> 「仕事でたとえます。チャットAI はメール文面を相談する相手、Claude Code は実際にファイルを開いて編集してくれる業務委託さんです。フォルダを覗き、ドラフトを作り、保存までする。だから触ってよい範囲とNG範囲を渡し、終わったら必ず差分レビューをする。この『差分レビュー』が、後ほど出てくる commit と相性が良いんです。」

【問いかけ】「今、自分が業務委託さんにお願いしている作業を1つ思い浮かべてください。それと同じ感覚で Claude Code に頼める、と考えてみてください。」
【日常のたとえ】「料理を頼んだら、最後に皿の中身を見るのと同じです。」
【仕事でのたとえ】「外注先に納品物をもらったら必ず差分や修正履歴をチェックしますよね。あれと同じです。」
【ここで混乱しやすい】「AI なんだから完璧だろう」と思わない。「業務委託さん」だと思ってチェックする。
【つなぎ】「では、よくある勘違いをまとめて潰しておきます。」

### スライド9：Claude Code のよくある勘違い

> 「よくある勘違いを5つ。魔法ではない、指示の質に比例します。自動で完璧にはならない。全部おまかせはトラブルのもと。最終チェックは人の仕事。仕様は更新が早いので公式で確認。この5つを心に留めてください。」

【問いかけ】「この中で『自分が誤解していた』と思うものはありますか？」
【日常のたとえ】「便利な家電を買っても、説明書を読まないと事故るのと同じです。」
【仕事でのたとえ】「新しい外注先と契約したら、初回は必ず納品物を細かくチェックしますよね。」
【ここで混乱しやすい】「AI に任せれば自分は楽できる」と思わないこと。「最終責任は自分」を強調する。
【つなぎ】「では章の最後、最初に覚えてほしい使い方です。」

### スライド10：Claude Code の最初に覚えるべき使い方

> 「明日から使える4ステップです。まず小さな作業を1つ任せる、触ってよいフォルダを限定する、完了後は差分を見る、違和感があれば即やめる。慣れてから範囲を広げる。これだけ守れば事故りません。」

【問いかけ】「自分の業務で『小さく試せる作業』を1つだけ思い浮かべてください。30秒、考える時間を取ります。」（30秒の沈黙時間を取る）
【日常のたとえ】「初めて来てもらう家事代行さんに、いきなり全部屋掃除を頼まないですよね。」
【仕事でのたとえ】「新人さんの初日タスクは、いきなり大型案件ではなく簡単な作業から渡しますよね。」
【ここで混乱しやすい】「いきなり大きな仕事を任せたい」気持ちを抑える。「小さく」が最大のコツ。
【つなぎ】「ここまでで Claude Code の正体は掴めました。次章は『普通のチャットAI と何が違うのか』をもう少し深掘りします。」

### 章のまとめの語り（20秒）

> 「ここだけ覚えて帰ってください。Claude Code は『作業してくれる人』。だから事前に範囲を決めて、終わったら差分を見る。これだけです。」

## 第2章：普通のチャットAI と何が違う？（スライド11〜14／所要7分）

### 章の冒頭の語り（30秒）

> 「第2章、一言でいうと『役割が違うので、両方使うのが強い』。チャットAI は考える支援、Claude Code は動く支援。単独完結ではなく、組み合わせて使う、という感覚を入れていきます。」

### スライド11：第2章「一言でいうと」

> 「もう一度まとめます。チャットAI は『考える支援』、Claude Code は『動く支援』。単独で完結しないので、組み合わせて使う。これを後ほど『指示文クッション方式』として整理します。今は『両方使うのが強い』という感覚だけ持ってください。」

【問いかけ】「皆さん、今日の業務で『考える』と『動く』、どちらの時間が長かったですか？」
【日常のたとえ】「献立を考えるのと、買い物に行って実際に作るのを、別の人が分担する感じです。」
【仕事でのたとえ】「企画を練る人と、それを資料化する人を分けるイメージです。」
【ここで混乱しやすい】「チャットAI で十分じゃない？」と思いがち。違いを次のスライドで体感してもらう。
【つなぎ】「日常のたとえで掴みましょう。」

### スライド12：日常のたとえ（料理の相談 vs 買い物・下ごしらえ）

> 「日常のたとえです。チャットAI は『今日の献立どうしよう』と相談する相手、話すだけです。Claude Code は冷蔵庫を開けて中身を見て、足りない食材を買いに行き、下ごしらえまでしてくれる人。同じ『料理』というテーマでも、担当する役割が違います。両方いて初めて食卓が完成します。」

【問いかけ】「皆さんの家の中で、献立を考える人と作る人、別ですか？ 同じですか？」
【日常のたとえ】「電話で相談する人と、家に来てくれる人。距離感が違います。」
【仕事でのたとえ】「壁打ちパートナーと、実装担当を分けるイメージです。」
【ここで混乱しやすい】「Claude Code 1つで全部できる」と思わないこと。「相談はチャット側」が原則。
【つなぎ】「仕事のたとえに置き換えます。」

### スライド13：仕事のたとえ（壁打ち vs 実ファイル操作）

> 「仕事のたとえです。チャットAI は企画を壁打ちする相手。Claude Code は資料テンプレを開いて埋めてくれる人。整理されたアウトプット、つまり『指示文』を作ってから Claude Code に渡すと、ぐっと結果が安定します。そしてその『指示文を整える』こと自体に、チャットAI が使えるんです。これが次の章のキーワード『指示文クッション方式』の予告です。」

【問いかけ】「自分の業務で『壁打ち相手が欲しい』と思う場面、どんな時ですか？」
【日常のたとえ】「友達に話してから本を書く、みたいな順番です。」
【仕事でのたとえ】「上司に方向性を確認してから資料を書き始めるのと同じ流れです。」
【ここで混乱しやすい】「整える時間がもったいない」と感じがち。実は整えるほうが結果として早い、と伝える。
【つなぎ】「では両者の使い分け早見表で締めます。」

### スライド14：両者の使い分け早見表＋「クッション方式」予告

> 「早見表で整理します。比較軸は5つ：用途／記憶／触れる範囲／向く相談／そして『どちらに頼むか』。チャットAI は相談・整形・要約、Claude Code は実ファイル作業。雑な要望はまずチャットAI で整える。これを『指示文クッション方式』と呼びます。詳しい手順は第8章で重点的に扱います。今は名前だけ覚えてください。」

【問いかけ】「自分の業務で『雑なまま頼んで失敗した』経験ありませんか？ それを次章で型にします。」
【日常のたとえ】「引っ越し業者に『いい感じで運んで』ではなく、荷物リストと搬入先を渡す、あの発想です。」
【仕事でのたとえ】「業務委託さんに口頭で依頼ではなく、依頼書を書いて渡すのと同じです。」
【ここで混乱しやすい】「チャットAI を経由するのが面倒」と感じる。実はこれが最大の時短になる、と強調。
【つなぎ】「次章は『依頼前の考え方』、つまりクッション方式の前段です。」

### 章のまとめの語り（20秒）

> 「ここだけ覚えて帰ってください。チャットAI と Claude Code は別物、両方を組み合わせて使うのが強い。」

## 第3章：作業を任せる前に必要な考え方（スライド15〜19／所要9分）

### 章の冒頭の語り（30秒）

> 「第3章、一言でいうと『依頼前の整理が成果の8割』です。任せる前に何をどう整理するかで、結果がほぼ決まります。ここで『指示文クッション方式』も軽く紹介します。」

### スライド15：第3章「一言でいうと」

> 「依頼前の整理が成果の8割を決めます。任せる前に『何を・どこまで』を決めておく。整理が雑だと結果も雑です。整理が苦手な人はチャットAI に整えてもらうのもアリ。整理の型は次のスライドで型を渡します。」

【問いかけ】「皆さん、依頼を受けるとき『何を、いつまでに、どこまで』が曖昧だとイラッとしませんか？ AI も同じです。」
【日常のたとえ】「家事代行さんに『適当に片付けて』と頼むのが一番危ない、あの感覚です。」
【仕事でのたとえ】「『この資料いい感じにまとめて』が一番事故るのと同じです。」
【ここで混乱しやすい】「整理に時間を取られる」と思わないこと。整理が結果として時短になる、と伝える。
【つなぎ】「日常のたとえで実感しましょう。」

### スライド16：日常のたとえ（「適当に片付けて」は危険）

> 「日常のたとえです。家事代行さんに『適当に片付けて』と頼むと、大事な書類まで捨てられかねません。事前に『触らない箱』と『捨ててよい袋』を決めて、完成形のイメージを渡し、途中で『ここどう？』と聞いてもらう。これだけで事故が激減します。」

【問いかけ】「『触らないでね』と先に言わなかったせいで、何かを失った経験ありますか？」
【日常のたとえ】「ホテルの忘れ物、思い出の品が捨てられたケース、よく聞きますよね。」
【仕事でのたとえ】「派遣初日に重要メールが整理フォルダに移されたあの事故、聞いたことあるはずです。」
【ここで混乱しやすい】「AI なら空気を読むだろう」と思わないこと。空気は読みません、明示が必須です。
【つなぎ】「仕事のたとえへ。」

### スライド17：仕事のたとえ（「いい感じに資料化」は危険）

> 「仕事の場面でも同じです。『この資料いい感じにまとめて』は曖昧の代表格。目的・読み手・分量・期日が抜けていると、ほぼ確実に期待と違うものが出てきます。整理してから渡せば、AI でも人でも一発で済みます。」

【問いかけ】「『いい感じに』と言われて困った経験、誰しもあると思います。皆さん、どんな場面でしたか？」

### スライド38：第8章 一言「雑な指示を直接投げない」

> 「雑な指示は事故のもとです。一度チャットAI で整える、これが指示文クッション方式の発想。本勉強会で一番伝えたいことです。次のスライドから具体的な手順に入ります。」

【日常のたとえ】「『適当に』が一番危ない、あの感覚。」
【仕事でのたとえ】「『いい感じに』が一番事故る、あの感覚。」
【ここで混乱しやすい】「クッション方式は面倒」と感じる。実は時短になる、と何度でも繰り返す。
【つなぎ】「6ステップを順に見ます。」

### スライド39：「指示文クッション方式」手順1〜6

> 「6つの手順です。①雑な要望をまず書き出す。②チャットAI に『指示文に整えて』と渡す。③出てきた指示文を人が一度レビュー。④Claude Code に渡す。⑤途中報告をチャットAI に貼って壁打ち。⑥次の指示文をチャットAI で作って渡す。⑤と⑥が分岐点での使い方です。1回で終わらないのがポイントです。」

【問いかけ】「6ステップ、難しそうに見えますが、実際は『雑→整える→渡す→受ける→壁打ち→次の指示』の繰り返しです。」
【日常のたとえ】「引っ越し業者に荷物リストを渡し、途中で『これどうする？』と相談しながら進める、あの流れ。」
【仕事でのたとえ】「外注先に依頼書を渡し、中間報告で軌道修正しながら最終納品まで導く、あの進め方。」
【ここで混乱しやすい】「①の段階で完璧な指示文を書こうとする」のは違います。雑なままでOK、整えるのは②。
【つなぎ】「では、なぜこの方式が効くのか、理由を整理します。」

### スライド40：クッション方式が効く理由

> 「効く理由は5つ。雑な要望を構造化してくれる、抜け漏れに気付ける、自分で書くより速い、途中報告の解釈も AI が手伝える、自分1人で詰まらない。特に最後、『1人で詰まらない』が大きいです。」

【問いかけ】「業務中、1人で抱え込んで時間を溶かした経験、ありませんか？ それを防ぎます。」
【日常のたとえ】「家族や友達に話したら頭が整理された、あの感覚を AI で再現する。」
【仕事でのたとえ】「壁打ち相手がいる安心感を、AI で常時手元に置く感覚。」
【ここで混乱しやすい】「クッション方式は『指示が下手な人のため』ではなく、上手な人ほど使うと効く。
【つなぎ】「では具体例3組、Before/After を見ます。」

### スライド41：良い/悪い指示×3組（資料／ファイル修正／開発作業）

> 「3組見ます。資料作成は『いい感じに』ではなく『対象◯◯／読み手△△／A4 1枚』。ファイル修正は『直しといて』ではなく『対象ファイル／変更箇所／触らない箇所』。開発作業は『機能足して』ではなく『目的／影響範囲／確認方法』。共通して5要素を入れる。これがクッション方式の出力イメージです。」

【問いかけ】「左の悪い指示、どこかで言った覚えありませんか？ 私もあります。」
【日常のたとえ】「『何か作って』ではなく『パスタ作って、辛さ控えめで』、あの違いです。」
【仕事でのたとえ】「『資料作って』ではなく『提案資料、対象◯◯、3枚以内』、あの違いです。」
【ここで混乱しやすい】「全部詳細に書け」ではなく、5要素を入れる。
【つなぎ】「次は分岐点での使い方、1往復で終わらせない運用です。」

### スライド42：分岐点での使い方（途中報告→壁打ち→次の指示）

> 「Claude Code は1回の指示で完結しません。途中報告が来たら、それをそのままチャットAI に貼って『ここどう判断する？』と壁打ち。次の指示文を作ってもらって、また Claude Code に渡す。この往復ループが効きます。」

【日常のたとえ】「家事代行さんから『この棚どうしますか？』と聞かれたとき、信頼できる人に相談してから答える、あの感じ。」
【仕事でのたとえ】「外注先からの中間報告を、上司や同僚に相談してから返答するのと同じ。」
【ここで混乱しやすい】「Claude Code に直接判断させたくなる」が、迷ったらチャットAI に壁打ち。
【つなぎ】「最後にもう一度、『1人で抱え込まない』を強調します。」

### スライド43：自分で無理に判断するより一度壁打ち

【問いかけ】「ここで5分間の体験タイムを取ります。手元のチャットAI（ChatGPT/Gemini/Claude.ai いずれでも可）を開いて、自分が今日やった『雑な指示』を1つ思い出して、AI に『指示文に整えて』と渡してみてください。整った指示文が返ってきたら、隣の方と『どこが変わったか』を共有してください。」（5分の体験タイムを取る）
【日常のたとえ】「友達に愚痴を聞いてもらったら頭が整理される、あの瞬間を AI で。」
【仕事でのたとえ】「同僚に『これってどう思う？』と聞いて頭を整える、あの瞬間。」
【ここで混乱しやすい】「壁打ちは時間の無駄」ではなく、結果として一番早い、と再強調。

## 想定問答（既存10問＋追加4問＝計14問）

### 既存10問

#### Q1. 「Claude Code を入れたいけど、自分のPCにインストールして大丈夫ですか？」
A. 会社支給 PC の場合は情シスに必ず確認してください。個人 PC でも、業務データを扱うなら所属部署のルールを確認してください。インストール自体は技術的には簡単ですが、社内ルールが優先します。

#### Q2. 「ChatGPT 使ってるから Claude Code はいらないのでは？」
A. 用途が違います。ChatGPT は『相談相手』、Claude Code は『手を動かす作業担当』。両方を組み合わせる『指示文クッション方式』が今日のメッセージです。

#### Q3. 「無料で使えますか？」
A. 利用プランや料金は更新があるので、各社公式情報で必ず確認してください。今日の勉強会では仕様の細部を断定しません。

#### Q4. 「セキュリティが心配。会社のデータを学習に使われませんか？」
A. 各社の利用規約とプラン設定により異なります。情シスに確認のうえ、業務利用ルールを定めてください。一般論として、機密情報は AI に渡す前にチーム合意が必要です。

#### Q5. 「コードを書けないと使えませんか？」
A. 書けなくても使えます。今日の勉強会自体が非エンジニア向けです。資料作成、データ整形、議事録要約など、コード不要の業務でも価値が出ます。

#### Q6. 「失敗したらどうやって戻すの？」
A. commit（セーブポイント）を使います。1機能・1修正＝1コミットの粒度で進めれば、戻したい時点に戻せます。詳細は第4章スライド24を参照。

#### Q7. 「Claude Code が勝手に色々書き換えるのが怖い」
A. 触ってよい範囲を CLAUDE.md と指示文で明示してください。違和感があれば即やめる、これが第1章スライド10の最初の使い方です。

#### Q8. 「日本語で指示して大丈夫ですか？」
A. 日本語で指示できます。ただし固有名詞や用語のブレは結果に影響するので、CLAUDE.md に用語集を入れておくと安定します。

#### Q9. 「memory と CLAUDE.md の使い分けが分かりません」
A. 判断軸は『毎回必要か』です。毎回必要なら CLAUDE.md、案件ごと・短期なら memory。詳細は第5章スライド29を参照。

#### Q10. 「skills を作ってみたい。何から始めればいいですか？」
A. まずは『3回以上繰り返している作業』を1つ書き出し、その手順を文章化してください。詳細仕様は公式情報で確認しつつ、社内の経験者と相談を。

### 追加4問

#### Q11. 「クッション方式って毎回やるのですか？」
A. 慣れてくると、簡単な依頼ではスキップしても構いません。ただし『初めての作業』『重要な作業』『過去に失敗した作業』では必ず使ってください。判断軸は『失敗した時の影響が大きいか』です。

#### Q12. 「ChatGPT/Gemini が手元になければクッション方式できないのですか？」
A. 厳密には『チャットAI が必須』ではなく、『指示を整える工程』が必須です。手元のチャットAI がなければ、紙に9項目チェックリストを書き出すだけでも代用できます。第3章スライド18の9項目を活用してください。

#### Q13. 「Claude Code の報告のどこからどこまでチャットAI に貼ればいいですか？」
A. 原則『全文』が安全です。短くしすぎると判断材料が落ちます。機密情報が含まれる場合は、その部分だけ伏字にして貼ってください。長すぎる場合は『状況サマリ＋直近のやり取り』に絞ります。

#### Q14. 「指示が長くなりすぎませんか？」
A. 長くなって構いません。長さより『5要素（目的・対象・触ってよい・触っちゃダメ・確認方法）』が入っているかが重要。慣れてくると、テンプレ化して使い回せるようになります。

## 講師運用メモ

- 5分体験タイム（スライド43）と演習（スライド44〜49）では『正解探し』をさせない。整え方そのものを体験させる。
- 章ごとの『ここで混乱しやすい』は、講師が一度自分の言葉で言い換えてから読むと刺さりやすい。

- 「アシスタントバイト君」「魔法の呪文」など幼稚な表現は使わない。「机に来て作業してくれるアシスタント」で統一。
- 90分版は時系列逆転なし。スライド番号順に進める。
