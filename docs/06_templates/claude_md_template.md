# CLAUDE.md テンプレート（業務スタッフ向け・穴埋め式）

<!--
このファイルは、自分のプロジェクト用に CLAUDE.md を作るための穴埋めテンプレートです。
使い方：
1. このファイルを自分の作業フォルダにコピーし、ファイル名を「CLAUDE.md」に変更する
2. 下の各セクションを、自分の業務に合わせて埋める
3. 迷ったら、セクションごとに書かれている「記入のヒント」や「記入例」を参考にする
4. 埋められない項目は「未定」と書いておく。空欄のままにしない

たとえ話：CLAUDE.md はこのプロジェクトの「取扱説明書」です。
初めて入った人が、これを読めば業務の前提が分かる状態を目指します。

注意：Claude Code の仕様は更新されることがあるため、
参照のされ方・配置場所などで不明点があれば公式情報で確認してください。
-->

## 6. 作業時の基本ルール

<!-- 記入のヒント：「いつ・何を・どうする」の形で箇条書きにする -->

（ここに記入）

記入例：
```
- 作業開始前に、input/ の最新CSVが当月分であることを確認する
- 数値は千円単位で丸める
- 前月比が±5%を超える項目は、必ずコメント欄に理由を添える
- 作業中は output/_作業中/ フォルダに保存し、完成後に output/ 直下に移す
```

## 7. ファイル命名規則

<!-- 記入のヒント：日付形式・区切り文字・禁止文字を明記 -->

（ここに記入）

記入例：
```
- 日付形式：YYYYMMDD（例：20260401）
- 区切り文字：アンダースコア「_」
- 例：月次レポート_202603.pdf、営業データ_20260401_v2.xlsx
- 禁止文字：スペース、全角記号、スラッシュ
- 「最終版」「final」の乱用禁止。バージョンは v1, v2 で管理
```

## 12. 日本語の文体ルール

<!-- 記入のヒント：ですます／である、敬語レベル、禁止表現を決める -->

（ここに記入）

記入例：
```
- 基本はですます調

```

## 13. Claudeへの注意事項（AIに渡す前提情報）

<!-- 記入のヒント：AIが誤解しそうなポイント、社内固有の事情を明記 -->

（ここに記入）

記入例：
```
- 「レポート」は月次営業レポートを指す。週報ではない
- 「田中さん」は営業部の田中さんを指す。経理部の田中さんではない
- 金額は税抜表記が原則。税込の場合は明示する
- 社外秘情報（取引先名・金額）はマスキングが必要な場合がある
- 迷ったら勝手に判断せず、必ず私に確認すること
```
