# フォルダ整理提案スキル

## このスキルは何をするか

対象フォルダ配下のファイル・サブフォルダ構成を読み取り、散らかっている箇所（命名規則違反、バージョン違いの重複、置き場所不明のファイル、空フォルダなど）を検出して、整理案を提案します。**このスキルはあくまで「提案」までで、実際の移動・削除・リネームは行いません。** 必ず人間が内容を確認し、承認してから別途実施します。

## いつ使うか

- プロジェクト終了後、フォルダが散らかってきたとき
- 引き継ぎ前の資料整理で、どこに何があるか見直したいとき
- 複数人で共有しているフォルダの運用ルールを立て直したいとき
- 年度・四半期ごとの定期整理タイミング

## 使ってはいけない場面

- 本番データ・顧客データが入ったフォルダ（誤削除リスクのため別プロセスで）
- Git 管理下のソースコードフォルダ（整理は git の履歴と整合させる必要あり）
- 権限が自分にないフォルダ（他者のファイルに触れるのは NG）

## 事前に準備するもの

- 対象フォルダのパス
- 命名規則（あれば）：プレフィックス、日付フォーマット、区切り文字など
- 保管ポリシー（あれば）：何ヶ月経過したら archive へ、など
- 「触ってはいけないフォルダ・ファイル」の除外リスト
- 整理案の承認者（誰の OK が出たら実行に移せるか）

## 手順

1. **対象フォルダの階層を読み取る**
   ツリー構造を把握し、サブフォルダ数・ファイル総数を把握する。
2. **ファイルを種別ごとに分類する**
   - ドキュメント（docx, pdf, md）
   - スプレッドシート（xlsx, csv）
   - 画像（png, jpg）
   - アーカイブ（zip, tar）
   - その他
3. **命名規則違反を検出する**
   - 日付が無い／フォーマットが揃っていない
   - 全角スペース・特殊記号・「コピー」などのシステム由来語が残っている
   - `final`, `final2`, `final_本当に最終` のような迷走ファイル
4. **重複候補を検出する**
   - 同名ファイルが異なる階層に存在
   - ファイル名のバージョン違い（`v1`, `v2`, `rev3` など）
   - サイズや日付が酷似
5. **未分類ファイルを検出する**
   - ルート直下にいるべきでないファイル
   - 「新規フォルダ」「Untitled」系の名前
   - 分類先が曖昧なファイル
6. **空フォルダ・ほぼ空フォルダを検出する**
7. **整理案を 3 レベルで提案する**
   - **レベル 1（安全）**：リネームのみ。ファイル移動なし
   - **レベル 2（移動）**：適切なサブフォルダに移動
   - **レベル 3（削除／アーカイブ）**：古いバージョン・明らかな重複を archive 送りに
8. **実行順序を提案する**
   レベル 1 → 2 → 3 の順で、段階的に承認・実施できる形にする。
9. **絶対に自動実行しない**
   提案はあくまで提案として提示し、ユーザーの承認後に実行ステップへ進む旨を明記する。

```
- 重複候補: X 件
- 未分類ファイル: X 件
- 空フォルダ: X 件

## 検出詳細

### 命名規則違反
| # | パス | 問題 | 推奨対応 |
| - | ---- | ---- | -------- |

### 重複候補
| # | ファイル A | ファイル B | 類似度 | 推奨対応 |

### 未分類ファイル
...

### レベル 1（安全・リネームのみ）
- X 件
- 所要時間目安: XX 分

### レベル 2（移動）
- X 件
- 所要時間目安: XX 分

### レベル 3（アーカイブ／削除候補）
- X 件
- 注意: 削除は必ず承認後に実行

1. レベル 1 のリネームを実施

4. 結果確認
5. レベル 3 を archive/ 配下に移動（削除ではなく退避を推奨）
```

## 品質チェック観点

- [ ] 実際の移動・削除を行っていない（提案のみ）
- [ ] 除外リストに含まれるファイルに言及していない
- [ ] 重複判定が名前だけでなく内容類似性の観点も考慮されている
- [ ] 推奨リネーム後のファイル名が命名規則に合っている
- [ ] 削除ではなく archive 送りを基本提案にしている（復元可能な形）
- [ ] 承認者が誰かが明記されている
- [ ] レベル分けされており、段階的に適用できる

## よくある失敗

- **勝手にファイルを動かす**：絶対 NG。提案まで。→ 本スキルは「見るだけ」。
- **「コピー」「新規フォルダ」を即削除提案する**：中身が業務上重要な場合がある。→ まず中身を確認し、退避（archive）を基本に。
- **命名規則を推測で決める**：既存のルールがあるはずなので、ユーザーに確認してから適用。
- **大量の提案を一気に出す**：数百件の提案は実行不能。→ レベル分け＋優先度を付けて提示。
- **隠しファイル・システムファイルを触ろうとする**：`.DS_Store`、`Thumbs.db` などは基本スルー。

## 使用例

### 入力例

> `~/Desktop/クライアント資料/A 社/` が 3 年分散らかっています。整理案を出してください。命名規則は `YYYYMMDD_案件名_種別_vN.拡張子` でお願いします。archive/ サブフォルダはまだ無いので、作るところから提案に含めてください。削除は絶対しないでください。

### 出力例

```
# フォルダ整理提案レポート

対象フォルダ: ~/Desktop/クライアント資料/A 社/
スキャン日時: 2026-04-24 10:00
総ファイル数: 312 件 / 総フォルダ数: 18 件

## 検出サマリー
- 命名規則違反: 84 件
- 重複候補: 23 組
- 未分類ファイル: 41 件
- 空フォルダ: 3 件

## 検出詳細

### 命名規則違反（抜粋 5 件、全 84 件）
| # | パス | 問題 | 推奨対応 |
| - | ---- | ---- | -------- |
| 1 | /提案書 最終版.docx | 日付なし／全角スペース | 20240315_A社提案_proposal_v1.docx（※日付は要確認） |
| 2 | /見積もり コピー.xlsx | 「コピー」残存 | 削除ではなく archive/ へ退避 |
| 3 | /議事録_final_final.docx | 迷走バージョン | 最新日付のもののみ残し、他は archive/ |
| 4 | /IMG_20240401.jpg | 種別がない | 20240401_A社提案_image_v1.jpg |
| 5 | /新規 Microsoft Word 文書.docx | 既定名のまま | 中身確認のうえリネームまたは archive/ |

### 重複候補（抜粋 3 組、全 23 組）
| # | ファイル A | ファイル B | 類似度 | 推奨対応 |
| - | --------- | --------- | ------ | -------- |
| 1 | /提案書.docx | /backup/提案書.docx | サイズ一致 | backup/ 側を archive/ に |
| 2 | /見積書_v1.xlsx | /見積書_v2.xlsx | 名前類似 | v1 を archive/ に |
| 3 | /議事録_20240310.docx | /会議メモ_0310.docx | 日付一致 | 中身確認後、片方を archive/ に |

### 未分類ファイル（抜粋 3 件、全 41 件）
- /メモ.txt（内容不明）
- /スクリーンショット 2024-04-01 12.34.56.png
- /Untitled.numbers

### 空フォルダ
- /old/
- /新規フォルダ/
- /tmp/

## 整理案

### レベル 1（安全・リネームのみ、84 件）
- 命名規則違反のうち、日付が確定しているもの 60 件を先にリネーム
- 残り 24 件は日付不明のため、ユーザー確認が必要

### レベル 2（移動、64 件）
- archive/ フォルダを新規作成（まだ存在しない）
- 旧バージョン・重複候補を archive/YYYY/ 配下に退避
- 未分類ファイルは /misc/ サブフォルダを新設して一旦退避

### レベル 3（アーカイブ、なし）
- 削除はご指示どおり行いません。すべて archive/ への退避で対応します。

## 実行手順（承認後）
1. archive/ フォルダ・misc/ フォルダを新設
2. レベル 1 のリネーム 60 件を実施
3. 結果を目視確認
4. レベル 2 の移動 64 件を実施
5. 日付不明 24 件についてユーザー確認・個別対応
6. 空フォルダは最後に（archive/empty_folders/ に移動するか残すかを相談）

## 重要事項
- 本提案の実行は承認者（中野さん）の OK を得てから行います
- 削除操作は含みません。すべて退避（archive）ベースです
- 除外指定があればお知らせください
```
