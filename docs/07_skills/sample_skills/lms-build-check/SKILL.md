# LMS 投入データ ビルド確認スキル

## このスキルは何をするか

LMS に投入する直前のデータ一式（講座情報の CSV、教材 PDF・動画、サムネイル画像など）を対象に、投入エラーや表示崩れにつながる不備を事前に検出します。投入後に発覚すると差し戻し対応が大きな手戻りになるため、**投入前の最終関所**として使います。

## いつ使うか

- LMS に新規講座データを投入する直前
- 既存講座の教材を差し替える直前
- 外部パートナーから受け取ったコンテンツを取り込む前の検収時
- LMS 仕様変更（項目追加など）に伴い、既存データを再投入するとき

## 使ってはいけない場面

- 投入先 LMS の仕様が未確定の段階（チェック基準が定まらないため）
- LMS への直接書き込みや削除操作（本スキルはあくまで「確認」のみ）
- 著作権・権利関係のチェック（法務・編集部門の確認が必要）

## 事前に準備するもの

- 投入予定のデータ一式が格納されたフォルダのパス
- LMS 仕様書（ファイル命名規則、必須項目、推奨サイズ、対応文字コードなど）
- 既存データのサンプル（比較対象として）
- 問題があった場合の差し戻し先（担当部門・担当者）

## 手順

1. **対象フォルダの全体構成を確認する**
   サブフォルダ、ファイル数、ファイル種別（CSV／PDF／MP4／PNG 等）を一覧化する。
2. **ファイル命名規則をチェックする**
   - 規定のプレフィックス／サフィックスがあるか（例：`course_001_thumbnail.png`）
   - 全角・半角、大文字・小文字の混在がないか
   - スペース・特殊記号（`＃`, `／`, `:`, スペースなど）が含まれていないか
3. **CSV の必須項目をチェックする**
   - ヘッダー行が仕様と一致しているか
   - 必須列に空欄がないか
   - ID の重複がないか
   - 日付フォーマットが統一されているか（例：`YYYY-MM-DD`）
4. **文字コードをチェックする**
   - CSV・テキストが UTF-8（BOM の有無は仕様に合わせる）になっているか
   - 改行コードが仕様どおりか（CRLF / LF）
5. **画像サイズ・容量をチェックする**
   - サムネイルは規定解像度（例：1280 × 720）か
   - ファイルサイズが上限を超えていないか
   - アスペクト比が崩れていないか
6. **動画・PDF の仕様確認**
   - 動画のコーデック・ビットレート・長さが仕様内か
   - PDF がパスワード保護されていないか、フォント埋め込みがされているか
7. **相互参照の整合性をチェックする**
   - CSV 内で指定している画像ファイル名が実在するか
   - リンク切れ（存在しないファイルを参照）がないか
8. **チェック結果をレポート形式でまとめる**
   「OK 項目」「NG 項目」「要確認項目」に分けて提示する。
9. **NG があった場合は修正案を提示する**
   ただし自動修正は行わず、担当者に判断を仰ぐ。

## 成果物の形式

```
# LMS 投入データ ビルド確認レポート

対象フォルダ: <パス>
確認日時: YYYY-MM-DD HH:MM
確認者: Claude + <担当者名>

## サマリー
- 総ファイル数: XX
- OK: XX / NG: XX / 要確認: XX

## NG 項目
| # | ファイル | 問題 | 推奨対応 |
| - | -------- | ---- | -------- |
| 1 | ... | ... | ... |

## 要確認項目
（判断が分かれる・仕様書に記載がない項目）

## OK 項目（カテゴリ別）
- 命名規則: すべて OK
- 文字コード: すべて UTF-8
- ...
```

## 品質チェック観点

- [ ] 仕様書に記載のチェック項目をすべてカバーしている
- [ ] NG と要確認が明確に区別されている
- [ ] 指摘に具体的なファイル名・行番号がある
- [ ] 推奨対応が示されている
- [ ] 自動修正や自動削除をしていない（確認のみ）
- [ ] レポートが担当者にそのまま共有できる粒度になっている

## よくある失敗

- **仕様書を読まずにチェックを始める**：仕様が LMS バージョンで異なる場合がある。→ 必ず最新仕様書を確認してから開始。
- **件数が多いと目視で流してしまう**：サンプリングせず全件チェックする仕組みにする。
- **「なんとなく OK」にしてしまう**：判断に迷ったら「要確認」に入れ、担当者に判断を仰ぐ。
- **自動で直してしまう**：ファイル名のリネーム等は、必ず人間の承認後に行う。本スキルは確認のみ。

## 使用例

### 入力例

> `/data/lms/2026q2_upload/` に来期投入予定のデータを置きました。LMS 仕様書は `/docs/lms_spec_v3.pdf` です。投入前チェックをお願いします。

### 出力例

```
# LMS 投入データ ビルド確認レポート

対象フォルダ: /data/lms/2026q2_upload/
確認日時: 2026-04-24 14:20
確認者: Claude + 中野

## サマリー
- 総ファイル数: 128
- OK: 120 / NG: 5 / 要確認: 3

## NG 項目
| # | ファイル | 問題 | 推奨対応 |
| - | -------- | ---- | -------- |
| 1 | course_012_Thumbnail.png | 命名規則違反（大文字混在） | course_012_thumbnail.png にリネーム |
| 2 | courses.csv (行 45) | 必須列「開始日」が空欄 | 担当者に開始日を確認して追記 |
| 3 | courses.csv | 文字コード Shift_JIS | UTF-8 (BOM なし) に変換 |
| 4 | course_020.pdf | パスワード保護あり | 保護解除後に再提出 |
| 5 | course_030_thumb.png | 解像度 640 × 360（仕様: 1280 × 720） | 再書き出し |

## 要確認項目
- course_099.mp4: 長さ 62 分（仕様上限 60 分と近接、要判断）
- courses.csv 行 120: カテゴリに仕様書に記載のない値「AI/DX」
- 画像 course_105_thumbnail.png: アスペクト比 16:10（仕様は 16:9）

## OK 項目
- 命名規則: 5 件を除き全て OK
- 文字コード: 1 件を除き UTF-8
- PDF: 1 件を除きフォント埋め込み済
- 動画コーデック: 全て H.264
- CSV 必須列: 1 件を除き欠損なし
- ID 重複: なし
- 画像参照切れ: なし
```
