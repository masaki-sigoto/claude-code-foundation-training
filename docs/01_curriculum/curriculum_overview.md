# Claude Code基礎勉強会 カリキュラム概要

本ファイルは、勉強会の全体設計書です。講師・受講者・自習者が、全体像と進め方を把握するためのものです。

---

## 1. 講座タイトル

**Claude Code基礎勉強会 〜なんとなく使うから、仕事を任せられる状態へ〜**

---

## 2. 対象者

- 非エンジニアの業務スタッフ
- 具体例：資料作成担当、ファイル整理担当、マニュアル作成担当、LMS（学習管理システム）用データ作成担当、ブログ記事構成担当
- Claude Codeを導入済みで、なんとなく使っているが、「同じ説明を毎回している」「指示が長くなる」と感じている人

---

## 3. 前提知識

特別な知識は不要です。

- プログラミング知識：不要
- ターミナル知識：不要
- 必要なのはPCの基本操作（ファイルの保存、フォルダの移動、文字入力）ができること
- Claude Codeが手元のPCに導入済みであること（未導入の場合は、事前に導入のみ済ませておく）

---

## 4. 到達目標

受講後、以下の10項目ができるようになることを目指します。

1. Claude Codeとは何か、通常のチャットAIと何が違うかを自分の言葉で説明できる
2. CLAUDE.md（プロジェクトの取扱説明書にあたるファイル）に何を書くべきかが分かる
3. memory（作業しながらClaudeが覚えた注意点）との使い分けが分かる
4. skills（繰り返し作業の専用レシピ）の役割が分かる
5. MCP（外部ツールとの接続口）がどういうものかのイメージがつく
6. 作業依頼の基本形（目的・対象・完了条件）を書ける
7. 作業前にフォルダ構成を整えてから依頼する習慣がつく
8. Claude Codeに「作業前の計画」をまず出してもらう進め方ができる
9. 事故りやすい指示（曖昧・範囲無限定・破壊的）を避けられる
10. 自分の業務をClaude Codeに任せるための、小さな手順書を自分で作れる

---

## 4-2. 教育方針：3段階説明の型

本カリキュラムでは、抽象的な技術用語で受講者を迷わせないために、**「日常 → 仕事 → Claude Code」の3段階**で各章を説明する方針を採ります。

- **理由1**：非エンジニア業務スタッフは、いきなり「エージェント」「コンテキスト」「フック」などの専門用語を聞いても、意味と使いどころがつながりません
- **理由2**：「自分の生活」と「自分の仕事」に置き換えてから「Claude Codeの世界」に橋渡しすることで、概念が記憶に残ります
- **理由3**：講師ごとに比喩がブレないよう、共通の比喩セット（後述）を固定します

### 6ステップ説明型（各章で必ずこの順で説明）

| ステップ | 内容 | ねらい |
|---|---|---|
| 1 | まず一言で言うと | 概念を1文で言い切る |
| 2 | 日常でたとえると | 家事・買い物・引っ越しなど、身近な場面で例える |
| 3 | 仕事でたとえると | 受講者の業務（資料作成・ファイル整理など）に近い場面で例える |
| 4 | Claude Codeではこうなる | 実際のClaude Codeでの使い方・呼び方に接続する |
| 5 | よくある勘違い | 受講者が陥りがちな誤解を先回りで潰す |
| 6 | 最初に覚えるべき使い方 | 最低限これだけ覚えればOK、という1ポイントを示す |

### 共通の比喩セット（揺らさない）

各章・各教材で以下の比喩を**固定**します。講師は揺らさないでください。

- **Claude Code** ＝ 作業してくれるAIエージェント／実際に机に来て作業してくれるアシスタント
- **CLAUDE.md** ＝ プロジェクトの取扱説明書／家事代行さんに渡す家のルール
- **memory** ＝ 作業しながら覚えた注意点／何度か来るうちに覚えた相手の好み
- **skills** ＝ 繰り返し作業の専用レシピ／料理のレシピ
- **MCP** ＝ 外部ツールとの接続口／外の倉庫・銀行・別の部屋へアクセスする鍵
- **hooks** ＝ 決まったタイミングで動く自動スイッチ／玄関を出る前の鍵・財布チェック
- **subagents** ＝ 専門作業を任せる別担当AI／引っ越しの荷造り担当・掃除担当

## 4-3. 重要運用：指示文クッション方式

本カリキュラムの**核となる運用コンセプト**です。第3章で導入し、第8章で重点的に扱います。

### 概念

雑な要望（人間） → **ChatGPT/Claude.ai/Gemini等の通常チャットAI**で整形 → 整った指示文を**Claude Code**へ貼る、という流れを徹底します。
通常チャットAIを「指示文を整える緩衝材（クッション）」として挟むため、**「指示文クッション方式」**と呼びます（名称は固定）。

### 日常／仕事でのたとえ

- **引っ越し業者**に「いい感じに運んで」と言うのではなく、**荷物リストと搬入先**を渡す
- **美容師**に「いい感じに切って」と言うのではなく、**写真と希望の長さ・雰囲気**を見せる
- どちらも「人間の頭の中の雑な要望を、相手が動ける形に整えてから渡す」点が共通

### なぜ初心者に効くか

1. **事故率が下がる**：雑な指示のまま投げると、Claude Codeが範囲を広く解釈して大量変更や意図しない削除を起こすリスクが高まります
2. **作業範囲が明確になる**：チャットAIに整形させる過程で、自分でも「結局何をやってほしかったのか」がはっきりします
3. **分岐点で迷わない**：Claude Codeの途中報告を読んで「次にどう指示すべきか」を自分一人で考えると判断ミスが起きやすい。ここをチャットAIに壁打ちさせます
4. **結果的に早く正確**：一見ステップが増えますが、やり直しが減るため総時間は短くなります

### 分岐点での使い方（壁打ちフロー）

1. Claude Codeから途中報告（「ここまでやりました」「次のどちらにしますか」など）が来る
2. その報告を**そのままコピー**して、ChatGPT/Claude.ai/Geminiに貼る
3. 通常チャットAIに「Claude Codeからこの報告が来た。次にどう指示すべきか、Claude Code向けの指示文として書いて」と頼む
4. 出てきた指示文をClaude Codeに貼って次工程へ

### 各章での扱い

- **第3章**（作業を任せる前の考え方）：概念を導入。「依頼前に整える」「分岐点で壁打ちする」の2場面を提示
- **第8章**（良い指示と悪い指示）：重点的に扱う。Before/After演習でクッション方式の効果を体感
- **その他の章**：折に触れて「ここはクッション方式で整える場面」と明示

注：本カリキュラム作成時点の運用ノウハウであり、Claude Codeや各チャットAIの公式機能ではありません。具体的なツール仕様は各サービスの公式情報で確認が必要です。

## 5. 全体構成（10章）

| 章 | タイトル | ひとことで | 各章のゴール |
|---|---|---|---|
| 第1章 | Claude Codeとは何か | 手を動かすAIエージェントの紹介 | 通常のチャットAIと「役割」が違うことを理解する |
| 第2章 | 通常チャットとの違い | なぜファイルに触れるのか | 「質問に答えるAI」と「作業するAI」の差を説明できる |
| 第3章 | 作業を任せる前の考え方 | 渡す前の準備が8割 | 依頼の前に整えておくべき3つのこと（目的・対象・完了条件）を言える |
| 第4章 | CLAUDE.md | プロジェクトの取扱説明書 | CLAUDE.mdに書くべき項目を思い出せる |
| 第5章 | memory | 作業しながら覚えた注意点 | CLAUDE.mdとmemoryの使い分けが説明できる |
| 第6章 | skills | 繰り返し作業の専用レシピ | skill化する候補を自分の業務から1つ挙げられる |
| 第7章 | MCP／hooks／subagents | 拡張機能の地図 | 3つの違いをイメージできる（細部は公式情報を確認） |
| 第8章 | 良い指示と悪い指示 | 事故らない依頼文の書き方 | 悪い指示を良い指示に直せる |
| 第9章 | 実際に任せてみる | 小さく試す | 自分の業務から1つ選び、実際に依頼文を書ける |
| 第10章 | チーム運用ルール | 一人で抱えないために | チーム内でCLAUDE.mdを共有する意義を語れる |

## 13. 進行上の注意

- **Claude Codeの仕様変更**：本カリキュラム作成時点と受講時点で、コマンドや画面が異なる可能性があります。当日は、公式情報で仕様を確認してから進めてください
- **演習の到達度は揃わない**：受講者の業務内容によって、演習の題材や進み方が変わります。「全員が完璧にできる」ことよりも、「全員が1つは持ち帰れる」ことを優先してください
- **危険操作は扱わない**：演習で扱うのは、削除を伴わない「新規作成・追記」中心の作業にします。フォルダ一括削除・一括置換などは本勉強会では扱いません
- **AI出力の確認を習慣化**：各演習の終わりに、「Claude Codeの出力を自分の目で確認する時間」を必ず設けてください
