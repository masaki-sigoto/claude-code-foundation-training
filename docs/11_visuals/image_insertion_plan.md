# 画像挿入計画マスター

このドキュメントは、Claude Code基礎勉強会教材における画像挿入計画の一覧です。
画像生成自体はNotebookLM等で別途行いますが、本ドキュメントで「どこに何の画像を入れるか」を一元管理します。

---

## 利用方針

- 初心者向け教材なので「分かりやすさ」を最優先（おしゃれより明瞭）
- 1枚に詰め込みすぎない（1画像1メッセージ）
- 日常のたとえと業務の置き換えが伝わる図を優先
- 技術用語だらけ・英語だらけの図は避ける
- 黒い画面（CUI）・コード断片・ロボット感の強い図は避ける（受講者は非エンジニア）
- すべての図は**日本語キャプション付き**で作る（英語のみのラベルはNG）
- 配色は落ち着いたトーン（mkdocs-material の indigo 系と相性が良い色）

---

## 画像タイプ分類

| タイプ | 用途例 |
|---|---|
| 概念図 | 抽象用語のイメージ化 |
| 比較図 | A vs B の違いを可視化 |
| フローチャート | 手順や流れ |
| 手順図 | 番号付きステップ |
| アイコン付き説明図 | 用語にアイコンを添える |
| before/after 図 | 改善イメージ |
| 関係図 | 用語間の関係 |
| チェックリスト図 | 確認項目 |
| 役割分担図 | 担当の住み分け |
| 業務イメージ図 | 実務シーン |
| 日常のたとえ図 | 比喩を絵で |
| チーム運用ルール図 | 運用フロー |

---

## ナンバリング規則

- 通し番号 `IMG-001` 〜 `IMG-040` を採番
- 章ごとに3〜5点を目安に配分
- 章内の順序はスライド出現順
- 別章で転用するときも番号は使い回さず、必要なら派生番号（例：IMG-005b）を作る

## 補足：画像番号の振り直し方針

教材改訂で画像が増減した場合：
- 既存番号は維持（番号変更は混乱のもと）
- 追加分は IMG-041 以降の連番で
- 廃止分は番号を欠番にし、本ドキュメントに「廃止」と注記
- 章単位で再編する場合のみ全体振り直しを検討

## 補足：NotebookLMへの共通指示テンプレ

各画像依頼の冒頭に以下を貼ると品質が揃いやすい：

```
前提：
- 対象は非エンジニアの業務スタッフ（初心者）
- 教材全体のトーンは落ち着いた信頼感（おしゃれより明瞭）
- 配色は indigo 系（mkdocs-material のデフォルト）と相性の良いトーン
- 文字は日本語、英語はサービス名等やむを得ないものに限る
- ロボット感の強いアイコン、黒い画面、コード断片は避ける
- 1枚に詰め込まず、1画像1メッセージを徹底
- 絵文字は使わない
```
