# MCP（Model Context Protocol）入門

業務スタッフ向け・初心者向けの説明資料です。
「MCPって何？」という段階の方を想定しています。

---

## 1. MCPとは何か

**MCP**は **Model Context Protocol** の略です。

一言でいうと、
**「Claude Codeが外の世界に手を伸ばすための接続口」**
です。

### たとえ話

Claude Code単体は、手元の机（ローカルのファイル）を触ることはできますが、
そのままでは「隣の部屋の書類棚（Google Drive）」や
「本社にある掲示板（Slack）」までは手が届きません。

MCPは、Claude Codeと外部サービスをつなぐ「廊下」や「扉」のようなものです。
この扉を通してはじめて、外部ツールや外部情報源とやりとりできるようになります。

### もう少し正確に

- MCPは、AIと外部ツール・外部データをつなぐための**共通の約束事（プロトコル）**
- この共通ルールに沿って作られた「MCPサーバー」を追加すると、
  Claude Codeはそのサーバー経由で外部サービスを呼び出せるようになる
- 対応しているサービスや具体的な提供機能は変化するため、
  **公式情報での確認が必要**

---

## 2. MCPがあると何ができるか（業務例）

MCPサーバーを接続することで、たとえば以下のようなことが可能になると言われています。

- Google Drive / Google Workspace のファイル読み取り
- GitHub の issue / PR の確認・作成
- Slack のメッセージ検索・投稿
- Notion のページ取得・更新
- カレンダーの予定確認
- 社内DBや社内ツールとの連携（設計次第）

※ 具体的な提供MCPの種類・機能は変わる可能性があります。
**実際に利用する際は、公式情報での確認が必要**です。

### 業務イメージ

- 「先週Slackに投稿された◯◯についての要件をまとめて」
- 「このリポジトリの未対応issueを一覧化して」
- 「Google Driveの◯◯フォルダにある議事録から論点を抽出して」

こうした「外部サービスを見に行ってまとめる」作業を依頼できるようになります。

## 3. MCPと通常のファイル操作の違い

| 項目 | 通常のファイル操作 | MCP経由の操作 |
| --- | --- | --- |
| 対象 | 自分のPC内のファイル | 外部サービス上のデータ |
| 必要なもの | ファイルパス | 接続設定・認証情報 |
| 権限 | OSのファイル権限 | 各サービスのアカウント権限 |
| リスク | ローカル範囲にとどまる | 外部への送信・書き込みが起こり得る |
| 速度 | 基本的に速い | ネットワーク経由で時間がかかる場合あり |
| 監査 | ローカルログ | サービス側のログも確認が必要 |

## 4. MCPを使う前に考えること

### 最重要：権限管理

- **誰のアカウントで接続しているか**
  - 個人アカウント？業務アカウント？
- **どこまで操作できる設定になっているか**
  - 読み取りのみ？書き込みもできる？
  - 全社共有のフォルダまで見える設定になっていないか？

### 認証情報の扱い

- APIキーやトークンを「ソース管理（Git）」にうっかり入れない
- 個人アカウントと業務アカウントを混在させない
- 使わなくなった認証情報は早めに失効させる

### データの持ち出しリスク

- MCP経由でやり取りされる情報は、**外部サービス側に送信される可能性**がある
- 顧客情報や機密情報をうっかり流さないよう注意

### 社内ルールとの整合

- 情報管理規定・セキュリティポリシーと矛盾しないか
- 「このサービスとの連携は許可されているか」を事前に確認
- 不明なら情シス・セキュリティ担当に相談

---

## 5. 初心者のおすすめ接し方

- 最初は**「概念を理解するだけ」で十分**
- 実際の接続作業は、詳しい人（エンジニア・情シス）と一緒に
- **「何でもつなげば良い」わけではない**
  - 本当に必要なものだけ
  - つなぐほど権限管理の手間も増える

業務スタッフとしては、まず
「MCPという接続口があって、使うと外部サービスと連携できる」
という理解ができていれば十分です。

---

## 6. よくある誤解

| 誤解 | 実際 |
| --- | --- |
| MCPがあれば何でもできる | できることは接続したMCPサーバーの機能次第。万能ではない |
| 接続すれば魔法のように連携する | 権限設定・認証・運用ルールの整備が必要 |
| 個人利用と業務利用は同じ感覚でOK | 違う。業務利用では情報管理ルールを必ず確認 |
| つなぐだけで安全 | データが外部送信される可能性があるため、接続範囲と権限の確認が必須 |
| 一度つないだら放置でいい | 認証情報・権限は定期的に見直す必要がある |

## 7. MCP導入チェックリスト（業務スタッフ向け）

接続・利用の前に、以下を確認してください。

- [ ] 接続先サービスは、業務利用が許可されているか
- [ ] 認証情報（APIキー・トークン）の保管場所は適切か
- [ ] どのデータが外部へ流れる可能性があるかを把握しているか
- [ ] 読み取り専用で足りるのに書き込み権限まで付与していないか
- [ ] 問題発生時にすぐ切断（無効化）する手順を知っているか
- [ ] 社内の情報管理規定・セキュリティポリシーに抵触しないか
- [ ] 個人アカウントと業務アカウントが混在していないか
- [ ] 接続後、ログや動作を確認する体制があるか

一つでも「？」がついたら、エンジニアや情シスに相談してください。

## 付録：用語メモ

- **MCPサーバー**：MCPの仕組みに沿って作られた「外部サービスと話すための窓口プログラム」
- **MCPクライアント**：MCPサーバーを呼び出す側（Claude Code側）
- **プロトコル**：通信のルール・約束事
